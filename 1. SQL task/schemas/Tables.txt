LOANS - таблица кредитных договоров

id_loan - ID договора;
dt_start - дата действия строк (начало);
dt_end - дата действия строк (конец);
id_client - ID клиента;
num_loan - номер договора;
dt_open_loan - дата открытия договора;
code_curr - код валюты договора;
int_rate - процентная ставка по основному долгу;
risk_group - группа риска;

##############################################################

CLIENTS - таблица клиентов

id_client - ID клиента;
dt_start - дата действия строк (начало);
dt_end - дата действия строк (конец);
name_client - наименование клиента;
type_client - тип клиента;
department - наименование подразделения клиента;

##############################################################

LOANS_FACT - таблица фактов по кредитным договорам за каждый день

id_loan - ID договора;
dt - дата факта (по состоянию на конец этого дня отображаются остатки по кредитам);
rest_od - остаток основного долга в валюте договора;
rest_od_eq - остаток основного долга в эквиваленте на BYN;
rest_pd - остаток просроченного долга в валюте договора;
rest_pd_eq - остаток основного долга в эквиваленте на BYN;

##############################################################

SCD-2, известный как медленно изменяющийся измерение типа 2, является стратегией, применяемой в хранилище данных для эффективного управления изменениями данных измерений. В типе 2 SCD, когда изменения происходят в атрибуте размеров, а не обновляют существующую запись в таблице измерений, новая запись вставлена для представления изменения. Этот подход сохраняет исторические данные, сохраняя как старые, так и обновленные версии информации.
